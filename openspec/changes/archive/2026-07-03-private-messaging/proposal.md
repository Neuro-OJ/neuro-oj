## Why

Neuro OJ 目前缺少用户间直接沟通的渠道。在 Hydro OJ 对比分析中，"站内信/通知"被列为 P0 核心缺失。用户需要一个简单的 1v1 文本私信功能来促进交流，但不需要专业 IM 级别的群聊、文件传输或端到端加密。

## What Changes

- 新增**私信系统**：允许注册用户之间发送和接收 1v1 文本消息
- 新增**会话管理**：每对用户自动共享一个会话，支持发送、列表、已读、删除
- 新增**实时推送**：基于现有 SSE 事件总线推送新消息通知，支持轮询降级
- 新增**消息已读状态**：追踪每个用户在每个会话中的最后阅读位置
- 新增**未读徽标**：导航栏显示未读消息总数
- 新增**用户入口**：用户主页添加"发送私信"按钮

## Capabilities

### New Capabilities
- `private-messaging`: 后端消息 API — 会话 CRUD、消息收发、已读追踪、SSE 事件推送、权限校验
- `message-ui`: 前端私信界面 — 会话列表页、聊天详情页、导航栏未读徽标、用户主页入口

### Modified Capabilities

- 无（新增功能，不修改现有能力的要求）

## Impact

- **noj-core**: 新增 4 张数据表、1 个服务文件、1 个路由文件、修改 event-bus 和 sse 路由
- **noj-ui**: 新增 1 个 composable、2 个页面文件、修改 Navbar 和用户主页
- **无第三方依赖新增**：全部基于现有 PostgreSQL + Redis + SSE 基础设施
